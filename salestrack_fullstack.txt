⏺ Đây là bài tập yêu cầu dùng Claude Code để xây dựng một web app full-stack.
  Tôi sẽ hướng dẫn bạn từng bước:

  ---
  Các bước hoàn thành bài tập

  Bước 1 — Chọn ý tưởng web app

  Chọn một app đơn giản nhưng có cả frontend lẫn backend, ví dụ:
  - Todo list app
  - Blog cá nhân
  - Ứng dụng quản lý chi tiêu
  - Chatbot nhỏ

  Bước 2 — Tạo repository GitHub

  1. Tạo repo mới trên GitHub (đặt là Public)
  2. Clone về máy: git clone <url>
  3. cd vào thư mục dự án

  Bước 3 — Dùng Claude Code để xây dựng app

  Mở Claude Code trong thư mục dự án và:
  1. Yêu cầu Claude Code lên kế hoạch trước: "Hãy tạo PLAN.md cho dự án web app 
  [tên app] full-stack"
  2. Yêu cầu Claude Code sinh code từng phần (backend API, frontend,
  database...)
  3. Yêu cầu tạo CLAUDE.md: "Hãy tạo file CLAUDE.md mô tả dự án này"
  4. Tạo README.md với hướng dẫn cài đặt và chạy app

  Bước 4 — Deploy app lên internet

  Chọn một trong các nền tảng miễn phí:
  - Vercel (frontend + serverless) — dễ nhất
  - Railway hoặc Render (full-stack)
  - Netlify (frontend) + Supabase (backend/DB)

  Bước 5 — Xuất file cuộc trò chuyện với Claude

  - Trong Claude Code: dùng lệnh /export hoặc lưu transcript cuộc hội thoại
  - Lưu file này vào trong repo (ví dụ: conversation.md hoặc claude-chat.json)

  Bước 6 — Push lên GitHub và nộp

  git add .
  git commit -m "Complete full-stack web app"
  git push origin main
  Gửi link GitHub public cho SEONGON.

  ---
  Checklist trước khi nộp

  - App chạy được và có URL public
  - Repo GitHub public
  - Có README.md
  - Có CLAUDE.md
  - Có PLAN.md (do Claude Code tạo)
  - Có file xuất cuộc trò chuyện với Claude

  ---
  Bạn muốn bắt đầu với ý tưởng app nào? Tôi có thể giúp bạn ngay từ bước lên kế
  hoạch.

⏺ Brainstorm: Web App Quản Lý Bán Hàng (Sales CRM)

  ---
  Tên gợi ý

  SalesTrack / DealFlow / QuanLyKhach

  ---
  Các module chính

  1. Quản lý Khách hàng

  ┌───────────────────────┬────────────────────────────────────────┐
  │       Thông tin       │                Chi tiết                │
  ├───────────────────────┼────────────────────────────────────────┤
  │ Họ tên, SĐT, Email    │ Thông tin cơ bản                       │
  ├───────────────────────┼────────────────────────────────────────┤
  │ Nguồn đến (source)    │ Facebook, Zalo, Giới thiệu, Website... │
  ├───────────────────────┼────────────────────────────────────────┤
  │ Ngày đầu tiên liên hệ │ Timeline tracking                      │
  ├───────────────────────┼────────────────────────────────────────┤
  │ Phân loại             │ Khách mới / Khách cũ / VIP             │
  ├───────────────────────┼────────────────────────────────────────┤
  │ Ghi chú               │ Sở thích, nhu cầu đặc biệt             │
  └───────────────────────┴────────────────────────────────────────┘

  2. Pipeline Tư vấn & Chốt đơn

  Trạng thái dạng Kanban board:
  Mới → Đang tư vấn → Báo giá → Chốt đơn ✅ / Từ chối ❌

  3. Theo dõi Đơn hàng

  - Lịch sử mua hàng của từng khách
  - Giá trị đơn hàng
  - Ngày mua → tính chu kỳ quay lại

  4. Dashboard Tổng quan

  - Tổng khách mới trong tháng
  - Tỷ lệ chốt đơn (conversion rate)
  - Doanh thu theo tháng
  - Khách sắp đến chu kỳ mua lại (reminder)

  ---
  Phân loại khách hàng tự động

  Khách MỚI     → chưa mua lần nào
  Khách CŨ      → đã mua ít nhất 1 lần
  Khách QUAY LẠI → mua lần 2 trở lên
  Khách VIP     → giá trị đơn hàng cao / mua nhiều lần

  ---
  Tech Stack gợi ý

  ┌──────────┬───────────────────────────────────────┐
  │  Layer   │               Công nghệ               │
  ├──────────┼───────────────────────────────────────┤
  │ Frontend │ React + Tailwind CSS                  │
  ├──────────┼───────────────────────────────────────┤
  │ Backend  │ Node.js + Express                     │
  ├──────────┼───────────────────────────────────────┤
  │ Database │ PostgreSQL (hoặc SQLite cho đơn giản) │
  ├──────────┼───────────────────────────────────────┤
  │ Deploy   │ Vercel (FE) + Railway (BE + DB)       │
  ├──────────┼───────────────────────────────────────┤
  │ Auth     │ JWT đơn giản                          │
  └──────────┴───────────────────────────────────────┘

  ---
  Tính năng ưu tiên (MVP)

  Phải có:
  - Thêm / sửa / xóa khách hàng
  - Cập nhật trạng thái tư vấn
  - Phân loại khách mới / cũ
  - Dashboard số liệu cơ bản

  Nâng cao (nếu có thời gian):
  - Export danh sách ra Excel
  - Nhắc nhở khách quay lại (reminder)
  - Tìm kiếm & lọc nhanh
  - Dark mode
